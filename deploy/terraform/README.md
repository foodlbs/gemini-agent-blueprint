# `deploy/terraform/` — Terraform module + operator runbook

Provisions everything `deploy.py` needs to land a parameterized
ReasoningEngine (display name set via `PROJECT_DISPLAY_NAME`). Per DESIGN.v2.md §10.

## What this module creates

| Resource | Why |
|---|---|
| GCS bucket `${PROJECT}-${TF_VAR_project_name}-assets` | Image/video asset hosting (public-read, 90-day lifecycle) |
| GCS bucket `${PROJECT}-${TF_VAR_project_name}-staging` | Vertex AI source tarball uploads from `deploy.py` |
| Secret `${TF_VAR_project_name}-github-token` | GitHub PAT for `repo_builder` |
| Secret `${TF_VAR_project_name}-telegram-bot-token` | Telegram bot token (read by app + bridge) |
| Secret `${TF_VAR_project_name}-telegram-webhook-secret` | Random 32-byte token for webhook verification (generated by Terraform) |
| SA `${TF_VAR_project_name}-app` | Agent Runtime runtime identity |
| SA `${TF_VAR_project_name}-telegram-bridge` | Cloud Run runtime identity |
| SA `${TF_VAR_project_name}-scheduler` | Cloud Scheduler OIDC identity |
| Project + bucket + secret IAM bindings | Per §10.3 |

## What this module does NOT create (out-of-band)

| Resource | Why | How to provision |
|---|---|---|
| Firestore database (Native mode) | One-time per project; usually pre-existing | Phase 1 below — console or one gcloud call |
| ReasoningEngine itself | Source-based deploy via Vertex SDK | `uv run python deploy.py` (Phase 3) |
| Telegram bridge Cloud Run | Container-based deploy | Phase 4 |
| Cloud Scheduler jobs | Depend on engine + bridge URLs | Phase 5 |

### Memory Bank — auto-attached, not separately provisioned

DESIGN.v2.md §10.4 + §15 Q2 described a separate `${TF_VAR_project_name}-memory`
instance provisioned via `gcloud ai memory-banks create`. **That
command does not exist** in gcloud SDK 565+. In ADK 2.0b1, Memory
Bank is an API surface on the ReasoningEngine itself — calls go to
`/reasoningEngines/{engine_id}/memories`. There is nothing to
provision separately.

`tools/memory.py` reads `GOOGLE_CLOUD_AGENT_ENGINE_ID` (auto-set
inside Agent Runtime) and constructs `VertexAiMemoryBankService`
against the engine's own ID. The role `roles/aiplatform.user` on
the project (granted to `${TF_VAR_project_name}-app` in `iam.tf`) covers Memory
Bank API access — no per-instance IAM binding needed.

## Phase 0 — Set your variables

This terraform module is parameterized via `var.project_name`. Set it (and the
secrets it consumes) before running anything below:

```bash
# Choose a short prefix for your resources. 2-14 chars, lowercase letters,
# digits, hyphens. This drives all SA / secret / bucket names.
export TF_VAR_project_name=gab            # default; rename for your fork

# Required GCP project (no default).
export TF_VAR_project="${GOOGLE_CLOUD_PROJECT}"

# Required secret-backed values (read once from .env or your shell).
export TF_VAR_github_token=$(grep -E '^GITHUB_TOKEN=' ../../.env | cut -d= -f2-)
export TF_VAR_telegram_bot_token=$(grep -E '^TELEGRAM_BOT_TOKEN=' ../../.env | cut -d= -f2-)
```

Set the matching env vars for `deploy.py` (Phase 3):

```bash
export PROJECT_DISPLAY_NAME=gemini-agent-blueprint
export PROJECT_APP_NAME=gemini_agent_blueprint
# TF_VAR_project_name doubles as PROJECT_PREFIX for deploy.py.
```

## Phase 1 — Enable Firestore (Native mode)

Skip if Firestore is already initialized in this project. Check:

```bash
gcloud firestore databases list --project=${GOOGLE_CLOUD_PROJECT}
```

If empty, create the default database:

```bash
gcloud firestore databases create \
  --project=${GOOGLE_CLOUD_PROJECT} \
  --location=us-west1 \
  --type=firestore-native
```

The bridge writes to a single collection `airel_v2_sessions` — no
schema setup needed (Firestore is schemaless).

## Phase 2 — `terraform apply`

```bash
cd deploy/terraform

# Pass secrets via env (never via -var on the command line — shell history).
export TF_VAR_project=${GOOGLE_CLOUD_PROJECT}
export TF_VAR_github_token=$(grep -E '^GITHUB_TOKEN=' ../../.env | cut -d= -f2-)
export TF_VAR_telegram_bot_token=$(grep -E '^TELEGRAM_BOT_TOKEN=' ../../.env | cut -d= -f2-)

terraform init
terraform plan -out=tfplan
# REVIEW THE PLAN. Expect: 26 resources to add, 0 to change, 0 to destroy.
terraform apply tfplan
```

After apply, capture the outputs:

```bash
terraform output  # human-readable summary
# Save the webhook secret separately — needed for setWebhook in Phase 5:
terraform output -raw telegram_webhook_secret_value
```

## Phase 4 — Build + deploy the Telegram bridge

```bash
# 1. Create Artifact Registry repo (one-time).
gcloud artifacts repositories create "${TF_VAR_project_name}" \
  --repository-format=docker \
  --location=us-west1 \
  --project=${GOOGLE_CLOUD_PROJECT}

# 2. Configure Docker auth.
gcloud auth configure-docker us-west1-docker.pkg.dev

# 3. Build + push.
IMAGE="us-west1-docker.pkg.dev/${GOOGLE_CLOUD_PROJECT}/${TF_VAR_project_name}/telegram-bridge:latest"
docker build -t $IMAGE telegram_bridge/
docker push $IMAGE

# 4. Deploy to Cloud Run.
ENGINE_ID=$(cat deploy/.deployed_resource_id)
BRIDGE_SA=$(cd deploy/terraform && terraform output -raw service_account_bridge)

gcloud run deploy "${TF_VAR_project_name}-telegram" \
  --image=$IMAGE \
  --region=us-west1 \
  --service-account=$BRIDGE_SA \
  --allow-unauthenticated \
  --ingress=all \
  --min-instances=0 \
  --max-instances=5 \
  --timeout=30s \
  --port=8080 \
  --set-env-vars="GOOGLE_CLOUD_PROJECT=${GOOGLE_CLOUD_PROJECT},FIRESTORE_DATABASE=(default),AGENT_RUNTIME_ENDPOINT=https://us-west1-aiplatform.googleapis.com/v1/${ENGINE_ID}" \
  --set-secrets="TELEGRAM_BOT_TOKEN=${TF_VAR_project_name}-telegram-bot-token:latest,TELEGRAM_WEBHOOK_SECRET=${TF_VAR_project_name}-telegram-webhook-secret:latest"

BRIDGE_URL=$(gcloud run services describe "${TF_VAR_project_name}-telegram" \
  --region=us-west1 --format='value(status.url)')
echo "Bridge URL: $BRIDGE_URL"
```

## Phase 5 — Cloud Scheduler + Telegram webhook registration

```bash
SCHEDULER_SA=$(cd deploy/terraform && terraform output -raw service_account_scheduler)

# Allow the scheduler SA to invoke the bridge Cloud Run.
gcloud run services add-iam-policy-binding "${TF_VAR_project_name}-telegram" \
  --region=us-west1 \
  --member="serviceAccount:${SCHEDULER_SA}" \
  --role="roles/run.invoker"

# Allow the scheduler SA to call streamQuery on the engine.
# (DESIGN.v2.md §10.3 lists `aiplatform.reasoningEngineUser` but that
# role is service-agent-only — use the broader aiplatform.user instead.)
gcloud projects add-iam-policy-binding ${GOOGLE_CLOUD_PROJECT} \
  --member="serviceAccount:${SCHEDULER_SA}" \
  --role="roles/aiplatform.user" \
  --condition=None

# Hourly trigger (PAUSED at create — operator unpauses after smoke test).
gcloud scheduler jobs create http "${TF_VAR_project_name}-hourly" \
  --location=us-west1 \
  --schedule="0 * * * *" \
  --time-zone=UTC \
  --uri="https://us-west1-aiplatform.googleapis.com/v1/${ENGINE_ID}:streamQuery?alt=sse" \
  --http-method=POST \
  --headers="Content-Type=application/json" \
  --message-body='{"class_method":"stream_query","input":{"user_id":"scheduler","message":{"role":"user","parts":[{"text":"Run a polling cycle."}]}}}' \
  --oidc-service-account-email=$SCHEDULER_SA \
  --oidc-token-audience="https://us-west1-aiplatform.googleapis.com/" \
  --max-retry-attempts=1
gcloud scheduler jobs pause "${TF_VAR_project_name}-hourly" --location=us-west1

# 15-min sweeper for HITL timeouts (ALSO paused initially).
gcloud scheduler jobs create http "${TF_VAR_project_name}-hitl-sweeper" \
  --location=us-west1 \
  --schedule="*/15 * * * *" \
  --time-zone=UTC \
  --uri="${BRIDGE_URL}/sweeper/escalate" \
  --http-method=POST \
  --oidc-service-account-email=$SCHEDULER_SA \
  --oidc-token-audience=$BRIDGE_URL \
  --max-retry-attempts=1
gcloud scheduler jobs pause "${TF_VAR_project_name}-hitl-sweeper" --location=us-west1

# Register the webhook with Telegram.
WEBHOOK_SECRET=$(cd deploy/terraform && terraform output -raw telegram_webhook_secret_value)
TELEGRAM_BOT_TOKEN=$(grep -E '^TELEGRAM_BOT_TOKEN=' .env | cut -d= -f2-)
curl -sS "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/setWebhook" \
  -d "url=${BRIDGE_URL}/telegram/webhook" \
  -d "secret_token=${WEBHOOK_SECRET}" \
  -d "allowed_updates=[\"callback_query\",\"message\"]"
```

## Smoke test (before unpausing the cron)

```bash
# Manually trigger one cycle.
gcloud scheduler jobs run "${TF_VAR_project_name}-hourly" --location=us-west1
```

Expected: a Telegram message lands in your $TELEGRAM_APPROVAL_CHAT_ID with Approve/Skip
buttons. Tap Approve → research runs → second message with the draft.

If both work end-to-end:

```bash
gcloud scheduler jobs resume "${TF_VAR_project_name}-hourly" --location=us-west1
gcloud scheduler jobs resume "${TF_VAR_project_name}-hitl-sweeper" --location=us-west1
```

## Tearing down (DESTRUCTIVE — be sure)

```bash
# Cloud Run + Schedulers
gcloud scheduler jobs delete "${TF_VAR_project_name}-hourly" --location=us-west1
gcloud scheduler jobs delete "${TF_VAR_project_name}-hitl-sweeper" --location=us-west1
gcloud run services delete "${TF_VAR_project_name}-telegram" --region=us-west1

# ReasoningEngine (deleting this also drops all Memory Bank facts —
# they live inside the engine).
ENGINE_ID=$(cat ../../deploy/.deployed_resource_id)
gcloud ai reasoning-engines delete $ENGINE_ID --region=us-west1

# Terraform-managed resources (assets bucket has force_destroy=false —
# delete manually first if you really mean it).
terraform destroy
```
